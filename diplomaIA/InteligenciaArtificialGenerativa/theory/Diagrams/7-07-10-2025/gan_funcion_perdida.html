📐 GANs - Función de Pérdida

El juego minimax entre Generador y Discriminador

🎯 Función de Pérdida General (Objetivo del Sistema)

FÓRMULA PRINCIPAL
minG maxD V(D, G) =

𝔼x~pdata(x)[log D(x)] + 𝔼z~p(z)[log(1 - D(G(z)))]

¿Qué significa esta expresión?

Es un juego minimax: el Discriminador (D) quiere maximizar esta función, mientras que el Generador (G) quiere minimizarla.

📊 Primer Término

𝔼x~pdata[log D(x)]

Significado: Esperanza del logaritmo de la predicción del discriminador sobre datos reales.

  • x: imagen real del dataset
  • D(x): probabilidad que D asigna a que x sea real
  • log D(x): se maximiza cuando D(x) → 1

Objetivo: D debe dar probabilidad alta (cercana a 1) a datos reales.

🎨 Segundo Término

𝔼z~p(z)[log(1 - D(G(z)))]

Significado: Esperanza del logaritmo de (1 - predicción) sobre datos falsos.

  • z: ruido aleatorio
  • G(z): imagen falsa generada
  • D(G(z)): probabilidad que D asigna a que G(z) sea real
  • log(1 - D(G(z))): se maximiza cuando D(G(z)) → 0

Objetivo: D debe dar probabilidad baja (cercana a 0) a datos falsos.

🔍 Entrenamiento del Discriminador (maximizar V)

DISCRIMINADOR
maxφ V(D, G) = 𝔼x~pdata[log D(x; φ)] + 𝔼z~p(z)[log(1 - D(G(z); φ))]

¿Qué hace el discriminador? Ajusta sus parámetros φ para:

  • Maximizar log D(x) para imágenes reales → quiere que D(x) ≈ 1
  • Maximizar log(1 - D(G(z))) para imágenes falsas → quiere que D(G(z)) ≈ 0
💡 Equivalencia con Binary Cross-Entropy: Esta función es equivalente a entrenar un clasificador binario con BCE Loss. Los datos reales tienen etiqueta 1 y los datos falsos tienen etiqueta 0.
En la práctica (BCE):

LossD = -[log D(x) + log(1 - D(G(z)))]

🎨 Entrenamiento del Generador (minimizar V)

GENERADOR - Versión Teórica
minθ V = 𝔼z~p(z)[log(1 - D(G(z; θ)))]

Nota importante: El primer término (datos reales) no depende de θ, por eso se elimina.

⚠️ Problema de Gradientes: Minimizar log(1 - D(G(z))) puede causar gradientes que desaparecen (vanishing gradients) al inicio del entrenamiento, cuando D es muy bueno y D(G(z)) ≈ 0.
GENERADOR - Versión Práctica
maxθ 𝔼z~p(z)[log D(G(z; θ))]

Solución: En lugar de minimizar log(1 - D(G(z))), se maximiza log D(G(z)).

  • Matemáticamente se comportan de manera similar
  • Pero los gradientes son más estables
  • El generador intenta que D(G(z)) → 1 (engañar al discriminador)
En la práctica (BCE con labels flipeados):

LossG = -log D(G(z))

Se pasan las imágenes generadas con etiqueta 1 (como si fueran reales)

📈 Visualización de las Funciones de Pérdida

Interpretación de log D(x):

  • Cuando D(x) = 1: log(1) = 0 (máximo)
  • Cuando D(x) → 0: log(0) → -∞ (penalización infinita)
  • Gradientes grandes cuando D(x) es pequeño

🔄 Algoritmo de Entrenamiento Alternado

Para cada época de entrenamiento:

  // Fase 1: Entrenar Discriminador (k veces)
  Para k iteraciones:
    1. Muestrear m ruidos: {z₁, z₂, ..., zₘ} ~ p(z)
    2. Muestrear m imágenes reales: {x₁, x₂, ..., xₘ} ~ p_data
    3. Calcular pérdida:
       Loss_D = -1/m Σ[log D(xᵢ) + log(1 - D(G(zᵢ)))]
    4. Actualizar φ con gradiente ascendente:
       φ ← φ + α · ∇_φ Loss_D

  // Fase 2: Entrenar Generador (1 vez)
    1. Muestrear m ruidos: {z₁, z₂, ..., zₘ} ~ p(z)
    2. Calcular pérdida (versión práctica):
       Loss_G = -1/m Σ log D(G(zᵢ))
    3. Actualizar θ con gradiente ascendente:
       θ ← θ + α · ∇_θ Loss_G
📌 Detalles importantes:
  • k: Número de veces que se entrena D por cada entrenamiento de G (típicamente k=1 o k=5)
  • m: Tamaño del batch (mini-lote)
  • α: Learning rate (puede ser diferente para D y G)
  • Alternancia: Se congela una red mientras se entrena la otra

🎯 Objetivo Final del Entrenamiento

El equilibrio ideal (equilibrio de Nash):

  • El generador aprende la distribución real: p_G = p_data
  • El discriminador no puede distinguir: D(x) = 0.5 para todo x
  • En este punto, ambas redes han alcanzado su mejor desempeño posible
✅ Señales de buen entrenamiento:
  • Las pérdidas de D y G se mantienen relativamente estables
  • Ninguna red domina completamente a la otra
  • Las imágenes generadas mejoran progresivamente
  • D(x_real) ≈ 0.7-0.8 y D(x_fake) ≈ 0.3-0.5
❌ Problemas comunes:
  • Mode Collapse: G genera siempre el mismo tipo de imagen
  • D demasiado fuerte: G no recibe gradientes útiles
  • G demasiado fuerte: D se confunde y deja de aprender
  • Inestabilidad: Las pérdidas oscilan salvajemente

🏗️ DC-GAN: Deep Convolutional GAN

Arquitectura con Redes Convolucionales para Generación de Imágenes

¿Qué es DC-GAN?

DC-GAN (Deep Convolutional GAN) es una variante de las GANs que utiliza redes neuronales convolucionales tanto en el generador como en el discriminador. Esta arquitectura fue propuesta en 2015 y mejoró significativamente la estabilidad del entrenamiento y la calidad de las imágenes generadas.

🎯 Ventajas principales:
  • Mejor estabilidad en el entrenamiento
  • Imágenes de mayor calidad
  • Representaciones latentes más significativas
  • Escalabilidad a imágenes de mayor resolución

🔸 Convoluciones Transpuestas

También llamadas "deconvoluciones", estas capas aumentan las dimensiones espaciales de la imagen.

  • Operación inversa a las convoluciones normales
  • Expande información de espacios pequeños a grandes
  • Permite generar imágenes desde vectores compactos

🔸 Batch Normalization

Se aplica después de cada convolución transpuesta (excepto la última).

  • Normaliza las activaciones en cada batch
  • Estabiliza el entrenamiento
  • Permite usar learning rates más altos
  • Reduce la dependencia de la inicialización

🔸 Función de Activación: ReLU

Se usa ReLU en todas las capas excepto la última.

  • ReLU(x) = max(0, x)
  • Introduce no-linealidad
  • Evita vanishing gradients
  • Computacionalmente eficiente

🔸 Capa de Salida: Tanh

La última capa usa Tanh para normalizar el output.

  • Tanh(x) ∈ [-1, 1]
  • Las imágenes se normalizan al rango [-1, 1]
  • Mejor que sigmoid para generación
  • Produce valores centrados en 0
class Generator(nn.Module):
    def __init__(self, latent_dim=100):
        super(Generator, self).__init__()

        # Capa inicial: de vector latente a tensor 3D
        self.fc = nn.Linear(latent_dim, 7 * 7 * 256)

        # Bloques de convolución transpuesta
        self.conv_blocks = nn.Sequential(
            # 7x7x256 -> 14x14x128
            nn.ConvTranspose2d(256, 128, kernel_size=5, stride=2, padding=2, output_padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(True),

            # 14x14x128 -> 28x28x64
            nn.ConvTranspose2d(128, 64, kernel_size=5, stride=2, padding=2, output_padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(True),

            # 28x28x64 -> 28x28x1
            nn.ConvTranspose2d(64, 1, kernel_size=5, stride=1, padding=2),
            nn.Tanh()
        )

    def forward(self, z):
        x = self.fc(z)
        x = x.view(-1, 256, 7, 7)  # reshape
        x = self.conv_blocks(x)
        return x

🔸 Convoluciones con Stride

Se usan convoluciones con stride=2 en lugar de max pooling.

  • Reduce dimensiones espaciales
  • Aprende su propio downsampling
  • Más eficiente que max pooling
  • Extrae características jerárquicas

🔸 Función de Activación: LeakyReLU

Se usa LeakyReLU en lugar de ReLU estándar.

  • LeakyReLU(x) = x si x > 0, else α·x
  • Típicamente α = 0.2
  • Evita "neuronas muertas"
  • Mejor flujo de gradientes

🔸 Sin Batch Norm en primera capa

La primera capa convolucional no usa Batch Normalization.

  • Evita que el discriminador aprenda solo por las estadísticas del batch
  • Fuerza al discriminador a aprender características reales
  • Mejora la estabilidad

🔸 Salida: Sigmoid

La capa de salida usa sigmoid para clasificación binaria.

  • σ(x) ∈ [0, 1]
  • 0 = imagen falsa
  • 1 = imagen real
  • Compatible con Binary Cross-Entropy
class Discriminator(nn.Module):
    def __init__(self):
        super(Discriminator, self).__init__()

        self.conv_blocks = nn.Sequential(
            # 28x28x1 -> 14x14x64
            nn.Conv2d(1, 64, kernel_size=5, stride=2, padding=2),
            nn.LeakyReLU(0.2),

            # 14x14x64 -> 7x7x128
            nn.Conv2d(64, 128, kernel_size=5, stride=2, padding=2),
            nn.BatchNorm2d(128),
            nn.LeakyReLU(0.2)
        )

        self.fc = nn.Sequential(
            nn.Flatten(),
            nn.Linear(7 * 7 * 128, 1),
            nn.Sigmoid()
        )

    def forward(self, x):
        x = self.conv_blocks(x)
        x = self.fc(x)
        return x

📋 Guías de Diseño de DC-GAN

✅ Recomendaciones

  • Usar convoluciones con stride en lugar de pooling
  • Usar Batch Normalization (excepto primera capa de D y última de G)
  • Eliminar capas fully connected (excepto en transiciones)
  • Usar ReLU en G (excepto salida que usa Tanh)
  • Usar LeakyReLU en D (todas las capas)
  • Normalizar imágenes al rango [-1, 1]

❌ Evitar

  • Max pooling o average pooling
  • Usar Sigmoid en capas ocultas
  • Demasiadas capas fully connected
  • Batch norm en la primera capa del discriminador
  • Learning rates muy altos
  • Entrenar D y G con la misma frecuencia sin monitoreo

🎛️ Hiperparámetros Típicos

  • Latent dim: 100
  • Batch size: 64-128
  • Learning rate (G): 0.0002
  • Learning rate (D): 0.0002
  • Optimizer: Adam
  • Beta1: 0.5
  • LeakyReLU alpha: 0.2

⚠️ Problemas Comunes

  • Mode Collapse: G genera siempre lo mismo
  • D demasiado fuerte: G no aprende
  • G demasiado fuerte: D se confunde
  • Gradientes inestables: Ajustar learning rates
  • Imágenes borrosas: Revisar arquitectura

💡 Consejos Prácticos para el Entrenamiento

🔧 Durante el desarrollo:

  • Monitorear las pérdidas: Loss_D y Loss_G deben mantenerse relativamente estables
  • Visualizar imágenes: Generar imágenes cada N épocas para ver progreso
  • Guardar checkpoints: El entrenamiento puede colapsar, guardar estados buenos
  • Normalización crucial: Imágenes en [-1, 1] con Tanh, o [0, 1] con Sigmoid
  • Batch size: Usar batches grandes (64-128) ayuda a estabilizar
  • Paciencia: DC-GANs pueden tardar muchas épocas en generar imágenes buenas

✨ Para mejores resultados:

  • Empezar con datasets simples (Fashion-MNIST, MNIST) antes de datasets complejos
  • Experimentar con el ratio de entrenamiento D:G (típicamente 1:1 o 5:1)
  • Usar data augmentation en imágenes reales
  • Considerar label smoothing (usar 0.9 en lugar de 1.0 para reales)
  • Probar diferentes inicializaciones de pesos

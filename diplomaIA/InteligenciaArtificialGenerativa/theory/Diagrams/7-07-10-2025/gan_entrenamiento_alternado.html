Fase 1: Entrenamiento del Discriminador

  • ✅ El generador está congelado (no actualiza sus pesos)
  • ✅ El discriminador recibe imágenes reales y falsas
  • ✅ D aprende a distinguir entre reales (etiqueta 1) y falsas (etiqueta 0)
  • ✅ Se actualiza solo el parámetro φ del discriminador

💻 Implementación en PyTorch

Fase 1: Entrenar Discriminador

# Congelar generador, entrenar discriminador
for param in generator.parameters():
    param.requires_grad = False

# Generar imágenes falsas
z = torch.randn(batch_size, latent_dim)
fake_images = generator(z).detach()  # detach() evita backprop a G

# Obtener imágenes reales
real_images = get_real_batch()

# Predicciones del discriminador
pred_real = discriminator(real_images)
pred_fake = discriminator(fake_images)

# Calcular pérdida (BCE Loss)
loss_real = criterion(pred_real, torch.ones_like(pred_real))
loss_fake = criterion(pred_fake, torch.zeros_like(pred_fake))
loss_D = loss_real + loss_fake

# Backpropagation y actualización
optimizer_D.zero_grad()
loss_D.backward()
optimizer_D.step()

Fase 2: Entrenar Generador

# Descongelar generador, congelar discriminador
for param in generator.parameters():
    param.requires_grad = True

for param in discriminator.parameters():
    param.requires_grad = False

# Generar nuevas imágenes falsas
z = torch.randn(batch_size, latent_dim)
fake_images = generator(z)

# El discriminador evalúa (sin actualizar sus pesos)
pred_fake = discriminator(fake_images)

# Calcular pérdida (queremos engañar a D, por eso usamos ones)
loss_G = criterion(pred_fake, torch.ones_like(pred_fake))

# Backpropagation y actualización
optimizer_G.zero_grad()
loss_G.backward()
optimizer_G.step()

🔑 Puntos Clave del Entrenamiento Alternado

  • Alternancia: Se entrena una red a la vez, manteniendo la otra congelada
  • Ratio k:1: Típicamente se entrena D más veces que G (ej: k=5 veces D, 1 vez G)
  • Learning rates diferentes: Generalmente lr_D < lr_G para evitar que D se haga demasiado fuerte
  • detach(): Al entrenar D, se usa .detach() en las imágenes generadas para no calcular gradientes hacia G
  • Labels flipeados en G: Se usan etiquetas "1" (real) para imágenes falsas al entrenar G
  • Balance crítico: Ninguna red debe dominar completamente a la otra

⚠️ Problemas Comunes en el Entrenamiento

  • Mode Collapse: G genera siempre el mismo output porque encontró una "trampa" que engaña a D
  • Vanishing Gradients: Si D es perfecto, G no recibe gradientes útiles para mejorar
  • Oscilaciones: Las pérdidas oscilan sin converger debido a la naturaleza adversaria
  • Inestabilidad: Cambios pequeños en hiperparámetros causan fallos completos

Binary Cross Entropy (BCE)

La función que permite usar perceptrones en modelos autorregresivos

El Problema del Perceptrón

🚫 Limitación del Perceptrón

Un perceptrón binario SOLO nos da la probabilidad de que el valor sea 1:

Perceptrón → P(xᵢ = 1 | x<i)

Pero nosotros necesitamos: Pθ(xᵢ = valor_real | x<i)

Es decir, necesitamos la probabilidad de que xᵢ tome el valor que realmente tiene (que puede ser 0 o 1).

Perceptrón
Red neuronal
↓
P(xᵢ = 1)
Solo una dirección

❓ Pregunta Clave

Si mi dato xᵢ es un 0, ¿cómo obtengo P(xᵢ = 0) si el perceptrón solo me da P(xᵢ = 1)?

Podríamos hacer: P(xᵢ = 0) = 1 - P(xᵢ = 1), pero esto no se puede usar directamente en la optimización de manera elegante.

La Solución: Binary Cross Entropy

✅ BCE resuelve el problema

Binary Cross Entropy es una función que "selecciona" automáticamente la probabilidad correcta según el valor real del dato.

Fórmula de BCE

BCE(xᵢ) = xᵢ · log(P(xᵢ=1|x<i)) + (1-xᵢ) · log(1-P(xᵢ=1|x<i))

¿Cómo funciona esta fórmula?

Es un "selector" inteligente que solo activa una de las dos partes dependiendo del valor real de xᵢ:

Si xᵢ = 0

El primer término se anula (0 × log...)

BCE = log(1 - P(xᵢ=1))

Obtenemos la probabilidad de que sea 0

Si xᵢ = 1

El segundo término se anula (0 × log...)

BCE = log(P(xᵢ=1))

Obtenemos la probabilidad de que sea 1

Descomposición de BCE

Primera Parte

xᵢ · log(P(xᵢ=1))

Función: Se activa cuando el valor real es 1

Penaliza: Si el modelo predice baja probabilidad cuando xᵢ=1

Segunda Parte

(1-xᵢ) · log(1-P(xᵢ=1))

Función: Se activa cuando el valor real es 0

Penaliza: Si el modelo predice alta probabilidad cuando xᵢ=0

Integración con el Modelo Autorregresivo

🔗 Conexión con la Clase Anterior

Recordemos que en el modelo autorregresivo queremos minimizar:

-log Pθ(x) = -Σ log Pθᵢ(xᵢ | x<i)

Reemplazando con BCE:

Para cada posición i en nuestra imagen (o dato):

-log Pθᵢ(xᵢ) = -BCE(xᵢ, Perceptrónθᵢ(x<i))

Donde Perceptrónθᵢ(x<i) es la red neuronal que recibe todos los píxeles anteriores y predice la probabilidad de que xᵢ = 1.

Ejemplo Concreto: Imagen de 3 Píxeles

Supongamos una imagen con 3 píxeles: [1, 0, 1]

Para x₁ = 1:

BCE₁ = 1 · log(Pθ₁(x₁=1)) + 0 · log(...)

Solo importa log(P(x₁=1))

Para x₂ = 0:

BCE₂ = 0 · log(...) + 1 · log(1 - Pθ₂(x₂=1|x₁))

Solo importa log(1 - P(x₂=1|x₁))

Para x₃ = 1:

BCE₃ = 1 · log(Pθ₃(x₃=1|x₁,x₂)) + 0 · log(...)

Solo importa log(P(x₃=1|x₁,x₂))

Pérdida Total:

Loss = -(BCE₁ + BCE₂ + BCE₃)

¿Por Qué BCE es una Función de Costo Local?

📍 Entendiendo "Local"

BCE es "local" porque calcula la pérdida para cada píxel individual (cada probabilidad condicional).

La pérdida global del dato completo es la suma de todas las BCE locales.

Proceso completo:

1. Para cada píxel i, calculamos BCEi (pérdida local)

2. Sumamos todas las BCEi para obtener la pérdida del dato completo

3. Promediamos sobre todos los datos de entrenamiento

4. Aplicamos descenso de gradiente para optimizar θ

Resumen de Conceptos Clave

📚 Puntos Esenciales

1. Problema: El perceptrón solo da P(xᵢ=1), pero necesitamos P(xᵢ=valor_real).

2. Solución: BCE actúa como selector automático según el valor real del dato.

3. Funcionamiento: BCE tiene dos términos, pero solo uno se activa dependiendo si xᵢ es 0 o 1.

4. Derivación: BCE surge de la necesidad de expresar Pθ(xᵢ=valor) de forma que podamos optimizar con perceptrones.

5. Conexión: Minimizar BCE es equivalente a minimizar la KL divergence entre nuestro modelo y la distribución real.

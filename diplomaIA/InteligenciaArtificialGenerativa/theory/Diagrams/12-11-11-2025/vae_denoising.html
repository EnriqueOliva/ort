🧹 VAE para Denoising (Eliminación de Ruido)

¿Qué es el Denoising?

Denoising es el proceso de eliminar ruido de imágenes. Una imagen con ruido tiene píxeles alterados (puntos aleatorios, distorsión) que no pertenecen a la imagen original.

El objetivo es: Imagen con ruido → Imagen limpia

🎯 El Problema

✨ Imagen Original

Imagen limpia y clara

⚠️ Con Ruido

Ruido gaussiano/salt-pepper

🎯 Objetivo

Recuperar imagen limpia

🔍 Tipos de Ruido

  • Ruido Gaussiano: Variaciones aleatorias en brillo/color (como estática de TV)
  • Salt-and-Pepper: Píxeles negros y blancos aleatorios (como sal y pimienta)

🏗️ Arquitectura del Sistema

🔧 Detalles de Implementación

🔽 Encoder

Función: Comprimir la imagen con ruido en una representación latente compacta.

  • Capas convolucionales
  • Reduce dimensionalidad
  • Aprende features importantes

⭐ Espacio Latente

Función: Representación abstracta y comprimida de la imagen.

  • Dimensión mucho menor
  • Captura esencia de la imagen
  • Ignora ruido (espera que lo haga)

🔼 Decoder

Función: Reconstruir imagen limpia desde el espacio latente.

  • Capas deconvolucionales
  • Aumenta dimensionalidad
  • Genera imagen sin ruido

📐 Función de Pérdida

Pérdida Compleja (según paper)

Loss = Loss_reconstruction + KL_divergence Pero Loss_reconstruction NO es simple ||x - x'||² En su lugar: Loss_reconstruction = ||x_clean_generada - x_clean_real||² + noise_term donde noise_term modela el ruido aprendido

El paper propone una función de pérdida más compleja donde el modelo no solo aprende a reconstruir la imagen, sino también aprende un modelo del ruido.

⚠️ Controversia en la Clase

El profesor cuestionó si esto es realmente NO supervisado. Para entrenar, necesitas pares: (imagen_con_ruido, imagen_limpia).

Los estudiantes defendieron que según el paper, el modelo aprende el proceso generativo del ruido y reconstruye comparando con la versión generada ruidosa, no directamente con la limpia.

🎓 Proceso de Entrenamiento

Pasos:

  1. Preparación de datos:
    • Tomar imágenes limpias del dataset
    • Generar ruido artificialmente (Gaussiano o Salt-Pepper)
    • Crear pares: (imagen_limpia, imagen_con_ruido)
  2. Forward pass:
    • Input: imagen con ruido
    • Encoder → espacio latente z
    • Decoder → imagen reconstruida
  3. Calcular pérdidas:
    • Reconstrucción: comparar con imagen limpia
    • KL Divergence: regularizar espacio latente
    • Opcional: términos adicionales de modelado de ruido
  4. Backpropagation:
    • Actualizar pesos del Encoder y Decoder
    • Optimizar para reconstruir imágenes limpias

🎯 Métricas de Evaluación

📊 PSNR

Peak Signal-to-Noise Ratio

PSNR = 10 · log₁₀(MAX²/MSE) MAX = valor máximo del píxel MSE = error cuadrático medio

PSNR alto = mejor calidad

🎨 SSIM

Structural Similarity Index

Mide similitud estructural entre imágenes. Considera luminancia, contraste y estructura.

SSIM cercano a 1 = mejor

👁️ Evaluación Visual

Observar visualmente las imágenes:

  • ¿Se eliminó el ruido?
  • ¿Se preservaron detalles?
  • ¿Hay artefactos nuevos?

💡 Desafíos y Consideraciones

🔍 Puntos Clave

  • Balance: Eliminar ruido SIN eliminar detalles importantes de la imagen
  • Tipos de ruido: El modelo debe generalizar a diferentes niveles/tipos de ruido
  • Supervisión: Aunque se llama "no supervisado", necesitas pares de entrenamiento
  • Complejidad: La función de pérdida es más compleja que un VAE estándar

✅ Ventajas del Enfoque VAE

  • Aprende representación compacta de imágenes
  • Puede interpolar en espacio latente
  • Framework probabilístico bien fundamentado
  • Potencial para generalizar a diferentes tipos de ruido

⚠️ Limitaciones

  • Requiere dataset con imágenes limpias para entrenar
  • Puede producir imágenes borrosas (típico de VAEs)
  • Función de pérdida compleja de implementar correctamente
  • Rendimiento depende del tipo de ruido en el que fue entrenado

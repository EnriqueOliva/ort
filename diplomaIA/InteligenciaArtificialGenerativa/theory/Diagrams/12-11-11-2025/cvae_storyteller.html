🎭 CVAE: Conditional VAE para Audio con Emoción

🎙️ StoryTeller - Narración Emocional con IA

Objetivo: Transformar texto en audio narrado con diferentes emociones.

El sistema toma un texto, lo convierte a voz neutral, y luego le aplica una emoción específica usando un Conditional VAE (CVAE).

😊 Las 4 Emociones

Calmada

😌

Tono suave, pausado, relajante. Para meditación, instrucciones.

Entusiasta

😄

Energética, alegre, motivadora. Para publicidad, noticias positivas.

Triste

😢

Melancólica, suave, emotiva. Para historias dramáticas.

Misterioso

🕵️

Intrigante, suspenso, bajo volumen. Para thrillers, misterio.

🎯 ¿Qué es un CVAE?

Conditional VAE = VAE + Condición

Un CVAE (Conditional Variational Autoencoder) es un VAE que recibe información adicional (una "condición") para controlar qué tipo de dato generar.

En este caso, la condición es la emoción deseada (calmada, entusiasta, triste, misteriosa).

🎼 Características de Audio (Features)

🎨 MFCC

Mel-Frequency Cepstral Coefficients

Capturan el timbre de la voz - la textura característica del sonido.

🎼 F0 (Pitch)

Frecuencia Fundamental

El tono de la voz - qué tan aguda o grave suena.

📢 RMS (Energy)

Energía/Intensidad

El volumen - qué tan fuerte o suave es el audio.

🔍 ¿Por qué no usar el audio crudo?

El audio crudo tiene dimensionalidad muy alta (miles de samples por segundo). Extraer estas features (MFCC, F0, RMS) comprime la información esencial del audio en una forma más manejable para la red neuronal.

🔑 Diferencia Clave: VAE vs CVAE

VAE Normal:

Encoder: x → z Decoder: z → x' No hay control sobre QUÉ se genera

CVAE (Conditional VAE):

Encoder: (x, c) → z ↑ ↑ audio emoción Decoder: (z, c) → x' ↑ ↑ latente emoción ¡PODEMOS CONTROLAR la emoción de salida!

✅ Ventaja del CVAE

Al condicionar tanto el encoder como el decoder con la etiqueta de emoción, el modelo aprende a separar las representaciones latentes por emoción. Así, el mismo contenido puede expresarse con diferentes tonalidades emocionales.

🎓 Entrenamiento del CVAE

Proceso:

  1. Dataset:
    • Clips de audio etiquetados por emoción
    • Ejemplo: "neutral", "feliz", "triste", "enojado", etc.
    • Cada audio tiene su etiqueta de emoción
  2. Extracción de Features:
    • De cada audio extraer: MFCC, F0, RMS
    • Estas features son el input real
  3. Forward Pass:
    • Concatenar features + one-hot emoción
    • Pasar por Encoder → μ, σ
    • Sampling: z = μ + σ·ε
    • Concatenar z + one-hot emoción
    • Pasar por Decoder → features reconstruidas
  4. Pérdidas:
    • Reconstrucción: ||features_orig - features_recon||²
    • KL Divergence: regularizar espacio latente
  5. Backpropagation:
    • Actualizar pesos de Encoder y Decoder

🚀 Uso en Inferencia

Una vez entrenado:

  1. Usuario escribe texto: "Hola, ¿cómo estás?"
  2. LLM (opcional) mejora el texto
  3. TTS (Coqui) convierte a audio neutral
  4. Extraer features del audio neutral (MFCC, F0, RMS)
  5. Usuario elige emoción: Entusiasta [0, 1, 0, 0]
  6. CVAE:
    • Encoder procesa features + emoción → z
    • Decoder genera features con emoción
  7. Vocoder convierte features → audio WAV
  8. ¡Audio final suena entusiasta! 😄🎙️

⚠️ Recomendaciones del Profesor

  • 🎯 Acotar duración: Empezar con palabras o frases cortas, NO historias largas
  • 📊 Buscar referencia: Encontrar un paper que haya hecho esto para tener baseline
  • 🔬 Validar viabilidad: Audio es complejo - tener expectativas realistas
  • 🎵 Problema de fase: Los espectrogramas pierden fase - puede haber ruido metálico

🎯 Desafíos Técnicos

  • Duración: Audios largos = espacios latentes enormes difíciles de entrenar
  • Vocoder: Convertir features → audio es no trivial (Griffin-Lim añade artefactos)
  • Dataset: Necesitas audio etiquetado con emociones (no tan común)
  • Evaluación: ¿Cómo medir si "suena triste"? Es subjetivo

📊 Evaluación

Métricas Propuestas:

  • Cualitativa: Escuchar y comparar audios generados vs originales
  • Análisis de espacio latente: ¿Las emociones se agrupan por separado?
  • Calidad espectral: Comparar espectrogramas input vs output
  • Modelo preentrenado: Usar modelo de clasificación de emociones para validar
  • Human Preference: Encuestas a personas evaluando naturalidad/emoción

✅ Puntos Fuertes del Proyecto

  • Aplicación práctica clara y tangible
  • Alto impacto potencial en múltiples dominios
  • Uso de CVAE (técnica avanzada) bien justificado
  • Pipeline completo end-to-end (texto → audio emocional)

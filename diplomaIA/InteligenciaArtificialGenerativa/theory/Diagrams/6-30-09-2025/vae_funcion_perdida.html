Los Dos Componentes

📸 Reconstruction Loss

Objetivo: Que la imagen reconstruida sea igual a la original.

L_recon = ||x - x̂||² (MSE)

o

L_recon = -∑ xᵢ log(x̂ᵢ) (BCE)

Intuición:

  • Mide la diferencia píxel a píxel
  • Penaliza reconstrucciones borrosas o incorrectas
  • MSE para valores continuos, BCE para binarios

Si solo usáramos esto: El modelo podría "memorizar" los datos sin aprender una representación útil.

📊 KL Divergence

Objetivo: Que la distribución del espacio latente sea similar a N(0, 1).

D_KL(q(z|x) || p(z))

Fórmula cerrada para Gaussianas:

= ½ ∑(μ² + σ² - log(σ²) - 1)

Intuición:

  • Regulariza el espacio latente
  • Empuja μ hacia 0 y σ hacia 1
  • Asegura que el espacio sea "suave" y continuo

Si solo usáramos esto: El modelo ignoraría los datos y produciría ruido aleatorio.

Visualización del Balance

Fórmula Completa (ELBO)

La pérdida del VAE es el negativo del ELBO (Evidence Lower Bound):

L(θ, φ) = -Eq[log pθ(x|z)] + DKL(qφ(z|x) || p(z))

Primer término:

Reconstrucción (log-likelihood)

Segundo término:

KL Divergence (regularización)

¿Por qué "ELBO"?

ELBO significa "Evidence Lower Bound". Maximizar el ELBO es equivalente a maximizar un límite inferior de la log-verosimilitud de los datos. Como queremos minimizar la pérdida, usamos el negativo del ELBO.

En Código (PyTorch)

def vae_loss(x, x_recon, mu, log_var):
    # Reconstruction loss (BCE o MSE)
    recon_loss = F.binary_cross_entropy(x_recon, x, reduction='sum')

    # KL Divergence: -0.5 * sum(1 + log(σ²) - μ² - σ²)
    kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())

    # Total loss
    return recon_loss + kl_loss

β-VAE: Una variante popular usa un factor β para controlar el peso del KL:

L = Reconstruction + β × KL

Con β > 1, se fuerza un espacio latente más estructurado (mejor para disentanglement).

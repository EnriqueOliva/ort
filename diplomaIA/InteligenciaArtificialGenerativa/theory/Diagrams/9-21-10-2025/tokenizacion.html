¿Qué es la Tokenización?

  • Proceso fundamental: Convierte texto (cadenas de caracteres) en números que las redes neuronales pueden procesar
  • Tokenizer: Herramienta que define cómo se divide el texto en unidades (tokens)
  • Bidireccional: Puede convertir texto → tokens y tokens → texto
  • Consistencia: Cada modelo usa su propio tokenizer, y debe ser el mismo para entrada y salida
  • Debe soportar TODO el texto posible: Cualquier palabra debe poder ser tokenizada

🔷 Tokenizer BERT

Mismo texto, diferentes tokens:

"Hola" → [1523]
"¿cómo" → [9821, 3344]

⚠️ Importante: Consistencia del Tokenizer

  • Un modelo = Un tokenizer: Si cambias el tokenizer, el modelo no funcionará
  • Números diferentes = Significados diferentes: Token 35 en GPT-2 ≠ Token 35 en BERT
  • Como el encoding de datos: Si entrenas con una codificación, debes usar la misma siempre
  • Puede causar errores graves: Incluso tokens que no existen en el vocabulario

Tokens Especiales

[PAD] - Relleno
[UNK] - Desconocido
[CLS] - Clasificación
[SEP] - Separador
[EOS] - Fin de Secuencia
[BOS] - Inicio de Secuencia

Cómo Funciona en la Práctica (GPT-2)

  • Paso 1: Cargar el tokenizer específico del modelo
  • Paso 2: inputs = tokenizer.encode("texto") → convierte a IDs numéricos
  • Paso 3: Pasar los IDs al modelo → model(inputs)
  • Paso 4: El modelo genera IDs de tokens de salida
  • Paso 5: tokenizer.decode(output_ids) → convierte de vuelta a texto legible

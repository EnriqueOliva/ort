Funcion de Perdida de las GANs

El Juego Minimax

La Idea Central: Un Juego

Las GANs son un juego entre dos redes neuronales:

GENERADOR (G)

El "falsificador"

Quiere crear imagenes
que parezcan reales

VS
DISCRIMINADOR (D)

El "detective"

Quiere detectar cuales
son falsas

Analogia: Es como entrenar a un falsificador de billetes y a un detector de billetes falsos al mismo tiempo. Cada uno mejora porque el otro mejora.

La Funcion Objetivo V(D,G)

V(D,G) = E[log D(x)] + E[log(1 - D(G(z)))]

El juego minimax:

min_G max_D V(D,G)

D quiere MAXIMIZAR V

Ser mejor detectando falsas

G quiere MINIMIZAR V

Ser mejor enganando a D

Las Dos Perdidas

Perdida del Discriminador

Objetivo: Clasificar correctamente reales y falsas.

Loss_D = BCE(D(real), 1) + BCE(D(fake), 0)

Quiere:

  • D(x_real) -> 1 (decir "real" para reales)
  • D(x_fake) -> 0 (decir "falsa" para falsas)

Importante: Usar .detach() en las imagenes falsas para no actualizar G.

Perdida del Generador

Objetivo: Enganar al discriminador.

Loss_G = BCE(D(fake), 1)

Quiere:

  • D(G(z)) -> 1 (que D crea que son reales)

Truco del Flipeo: Aunque son falsas, usamos etiqueta 1 para que G aprenda a enganar.

El Truco del Log

La formulacion original para G tiene un problema:

Original (problematico)

min log(1 - D(G(z)))

Cuando G es malo: D(G(z)) ~ 0
log(1-0) = 0 -> Gradiente PEQUENO
G no aprende cuando mas lo necesita

Mejorado (usado en practica)

max log(D(G(z)))

Cuando G es malo: D(G(z)) ~ 0
log(0.01) = -4.6 -> Gradiente GRANDE
G aprende rapido al inicio

Entrenamiento Alternado

No se entrenan juntos. Se alternan:

Codigo en PyTorch

# === ENTRENAR DISCRIMINADOR ===
real_labels = torch.ones(batch_size, 1)
fake_labels = torch.zeros(batch_size, 1)

z = torch.randn(batch_size, latent_dim)
fake_images = G(z)

optimizer_D.zero_grad()
loss_D_real = criterion(D(real_images), real_labels)
loss_D_fake = criterion(D(fake_images.detach()), fake_labels)  # .detach()!
loss_D = loss_D_real + loss_D_fake
loss_D.backward()
optimizer_D.step()

# === ENTRENAR GENERADOR ===
z = torch.randn(batch_size, latent_dim)
fake_images = G(z)

optimizer_G.zero_grad()
loss_G = criterion(D(fake_images), real_labels)  # Truco del flipeo!
loss_G.backward()
optimizer_G.step()

Resumen para el Parcial

V(D,G): E[log D(x)] + E[log(1-D(G(z)))] - D maximiza, G minimiza

Loss_D: BCE(D(real), 1) + BCE(D(fake.detach()), 0)

Loss_G: BCE(D(fake), 1) - el truco del flipeo

Entrenamiento: Alternado, lr_D < lr_G, usar .detach()

Equilibrio: Cuando G es perfecto -> D(x) = 0.5 para todo x

Funcion de Perdida de Language Models

Cross-Entropy / Negative Log-Likelihood

La Idea Central

El modelo aprende a predecir la siguiente palabra. La perdida mide que tan lejos esta de acertar.

Loss = -log(p_correcta)

Analogia: Es como un examen de opcion multiple donde el modelo tiene que elegir la siguiente palabra correcta entre miles de opciones.

Ejemplo Numerico

Entrada: "El gato come"

Palabra correcta: "pescado"

Logits del modelo:

"pescado": 2.8
"comida": 1.5
"raton": 0.3
...

Despues de Softmax:

"pescado": 0.40
"comida": 0.11
"raton": 0.03
...
Loss = -log(0.40) = 0.92

Si p_pescado fuera 0.90 → Loss = -log(0.90) = 0.11 (mucho mejor!)

El Comportamiento del Logaritmo

p → 0

Loss → infinito

Penaliza mucho los errores

p = 0.5

Loss = 0.69

Zona intermedia

p → 1

Loss → 0

Prediccion perfecta

Logits vs Probabilidades

LOGITS

Salida cruda del modelo

  • Pueden ser negativos
  • No suman 1
  • No son probabilidades
[2.5, -0.3, 1.8, 0.1]

PROBABILIDADES

Despues de Softmax

  • Entre 0 y 1
  • Suman 1
  • Son probabilidades!
[0.45, 0.03, 0.22, 0.30]

Softmax: p_i = exp(logit_i) / Sum(exp(logit_j))

La exponencial convierte todo a positivo, luego se divide por la suma para que todo sume 1.

Perplexity: La Metrica

Perplexity = exp(Loss_promedio)

Interpretacion: Mide que tan "confundido" esta el modelo.

Perplexity = 1

Perfecto

Siempre predice bien

Perplexity ~ 100

Bastante bueno

Modelo decente

Perplexity = V

Aleatorio

V = tamano vocabulario

Codigo en PyTorch

# Entrenamiento
def train_step(model, batch, optimizer):
    inputs, targets = batch

    # Forward: modelo produce logits
    logits = model(inputs)  # [batch, seq_len, vocab_size]

    # Perdida: Cross-Entropy
    loss = F.cross_entropy(
        logits.view(-1, vocab_size),  # [batch*seq_len, vocab_size]
        targets.view(-1)               # [batch*seq_len]
    )

    # Backward
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    return loss.item()

# Calcular Perplexity
perplexity = math.exp(avg_loss)

Resumen para el Parcial

Cross-Entropy: L = -log(p_correcta) - penaliza cuando p es bajo

Logits: Salida cruda, no son probabilidades, pueden ser negativos

Softmax: Convierte logits a probabilidades: exp(x_i) / sum(exp(x_j))

NLL: Negative Log-Likelihood = Cross-Entropy (equivalentes)

Perplexity: exp(Loss_promedio) - mas baja = mejor

Modelo determinístico: Misma entrada = misma distribucion; aleatoriedad viene del sampling

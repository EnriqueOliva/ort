### 1. All Albumentations Transforms

Albumentations provides over 100 transforms, categorized by functionality. Below is a comprehensive list based on the official documentation, grouped into key categories (geometric, color, blur, noise, and others). For each, I've included purpose, key parameters with recommended values (drawn from docs and benchmarks), use cases, cautions (especially for segmentation), and a code example. Recommendations are tailored for efficiency and effectiveness in segmentation tasks—e.g., lower probabilities (p=0.2-0.5) to avoid over-augmentation, and nearest-neighbor interpolation for masks to preserve edges.

#### Geometric Transforms
These alter spatial structure and must sync with masks/bboxes in segmentation.

| Name | Purpose | Parameters (Recommended) | Use Case | Caution (for Segmentation) | Code Example |
|------|---------|---------------------------|----------|----------------------------|-------------|
| Resize | Scales image to fixed size. | height/width (e.g., 384), interpolation=cv2.INTER_LINEAR (image), cv2.INTER_NEAREST (mask), p=1.0 | Standard preprocessing for model input. | None—always use NEAREST for masks to avoid pixel blending. | `A.Resize(384, 384)` |
| HorizontalFlip | Flips image left-right. | p=0.5 | Symmetric objects (e.g., humans facing camera). | Safe if subjects aren't directional (e.g., avoid for text). | `A.HorizontalFlip(p=0.5)` |
| VerticalFlip | Flips image up-down. | p=0.3 | Rare in natural images; use sparingly. | Avoid for upright subjects (e.g., people)—distorts anatomy. | `A.VerticalFlip(p=0.3)` |
| ShiftScaleRotate | Applies random shift, scale, rotation. | shift_limit=0.1, scale_limit=0.2, rotate_limit=15 (not 30), border_mode=cv2.BORDER_CONSTANT, p=0.5 | Simulates camera movement. | Limit rotation to ±15° for upright humans; excessive distorts proportions. | `A.ShiftScaleRotate(shift_limit=0.1, scale_limit=0.2, rotate_limit=15, p=0.5)` |
| ElasticTransform | Applies elastic deformations via grid. | alpha=120 (intensity), sigma=6 (smoothness), p=0.3 | Simulates soft tissue deformation (medical). | Avoid for human segmentation—blurs edges, distorts bodies unnaturally. | `A.ElasticTransform(alpha=120, sigma=6, p=0.3)` |
| GridDistortion | Distorts via random grid warp. | num_steps=5, distort_limit=0.3, p=0.3 | Non-rigid warps for robustness. | Use low params; can blur boundaries if too aggressive. | `A.GridDistortion(num_steps=5, distort_limit=0.3, p=0.3)` |
| OpticalDistortion | Barrel/pincushion distortion. | distort_limit=0.2, shift_limit=0.1, p=0.3 | Lens effects. | Moderate to preserve edges. | `A.OpticalDistortion(distort_limit=0.2, p=0.3)` |
| Perspective | Applies perspective warp. | scale=(0.05, 0.1), p=0.3 | Simulates viewpoint changes. | Safe but test for edge artifacts in segmentation. | `A.Perspective(scale=(0.05, 0.1), p=0.3)` |
| Affine | General affine transform (shear/translate/scale). | translate_percent=0.1, scale=0.2, shear=10, p=0.5 | Combined geometric changes. | Efficient alternative to ShiftScaleRotate; use NEAREST for masks. | `A.Affine(translate_percent=0.1, scale=(0.9, 1.1), p=0.5)` |
| RandomCrop | Crops random patch. | height/width=384, p=1.0 | Focus on regions of interest. | Ensure crops contain subjects; safe for segmentation. | `A.RandomCrop(384, 384)` |

#### Color Transforms
Pixel-level; don't affect masks. Ideal for mixed color/B&W datasets.

| Name | Purpose | Parameters (Recommended) | Use Case | Caution (for Segmentation) | Code Example |
|------|---------|---------------------------|----------|----------------------------|-------------|
| RandomBrightnessContrast | Adjusts brightness/contrast. | brightness_limit=0.2, contrast_limit=0.2, p=0.3 | Handles lighting variations. | None—enhances robustness without blurring. | `A.RandomBrightnessContrast(brightness_limit=0.2, contrast_limit=0.2, p=0.3)` |
| HueSaturationValue | Shifts hue/saturation/value (HSV). | hue_shift_limit=20, sat_shift_limit=30, val_shift_limit=20, p=0.3 | Color variations in portraits. | For B&W: set sat_shift_limit=0 to avoid artifacts. | `A.HueSaturationValue(hue_shift_limit=20, sat_shift_limit=30, p=0.3)` |
| RGBShift | Shifts R/G/B channels. | r_shift_limit=20, g/b=20, p=0.3 | Color jittering. | Safe; use for mixed color/B&W to simulate desaturation. | `A.RGBShift(r_shift_limit=20, p=0.3)` |
| ToGray | Converts to grayscale. | num_output_channels=3 (replicate gray to RGB), p=0.2 | Handles B&W images. | Apply conditionally; replicate channels for RGB models. | `A.ToGray(num_output_channels=3, p=0.2)` |
| CLAHE | Contrast-limited adaptive histogram equalization. | clip_limit=2.0, tile_grid_size=(8,8), p=0.3 | Extreme lighting (silhouettes). | Enhances edges; apply to LAB L-channel for color safety. | `A.CLAHE(clip_limit=2.0, p=0.3)` |

#### Blur Transforms
Pixel-level; avoid heavy use in segmentation.

| Name | Purpose | Parameters (Recommended) | Use Case | Caution (for Segmentation) | Code Example |
|------|---------|---------------------------|----------|----------------------------|-------------|
| GaussianBlur | Applies Gaussian blur. | blur_limit=3-7, p=0.2 | Simulates out-of-focus. | Avoid—blurs edges critical for boundaries. | `A.GaussianBlur(blur_limit=3, p=0.2)` |
| MedianBlur | Median filter blur. | blur_limit=3-5, p=0.1 | Noise reduction simulation. | Low limits only; preserves edges better than Gaussian. | `A.MedianBlur(blur_limit=3, p=0.1)` |
| MotionBlur | Simulates motion. | blur_limit=3-7, p=0.2 | Dynamic scenes. | Use sparingly—can smear boundaries. | `A.MotionBlur(blur_limit=7, p=0.2)` |

#### Noise Transforms
Pixel-level; match dataset noise (~3% grainy).

| Name | Purpose | Parameters (Recommended) | Use Case | Caution (for Segmentation) | Code Example |
|------|---------|---------------------------|----------|----------------------------|-------------|
| GaussNoise | Adds Gaussian noise. | var_limit=(10,30), p=0.1 | Simulates sensor noise. | Low var for clean datasets; doesn't blur edges. | `A.GaussNoise(var_limit=(10,30), p=0.1)` |
| ISONoise | Camera ISO noise. | intensity=(0.1,0.3), color_shift=(0.01,0.05), p=0.1 | Photography noise. | Matches ~3% grain; safe for edges. | `A.ISONoise(intensity=(0.1,0.3), p=0.1)` |

#### Other Transforms
Includes intensity, normalization, etc.

| Name | Purpose | Parameters (Recommended) | Use Case | Caution (for Segmentation) | Code Example |
|------|---------|---------------------------|----------|----------------------------|-------------|
| Normalize | Normalizes channels. | mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225], p=1.0 | Prepares for CNNs. | Always last; use per-image stats for regularization. | `A.Normalize(mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])` |
| RandomGamma | Applies gamma correction. | gamma_limit=(80,120), p=0.2 | Lighting/exposure simulation. | Good for silhouettes; preserves edges. | `A.RandomGamma(gamma_limit=(80,120), p=0.2)` |
| ChannelShuffle | Shuffles channels. | p=0.1 | Color robustness. | Rare use; safe but test for B&W. | `A.ChannelShuffle(p=0.1)` |

### 2. Segmentation-Specific Best Practices
- **Best Transforms**: Geometric (flips, crops, affine with low params); color (brightness/contrast, HSV for variations); noise (light GaussNoise). Use OneOf for variety: e.g., `A.OneOf([A.RandomBrightnessContrast(p=1.0), A.CLAHE(p=1.0)], p=0.3)`.
- **Transforms to Avoid**: ElasticTransform/GridDistortion (edge blurring); heavy blur (Gaussian/Motion); VerticalFlip (upright subjects). These introduce artifacts harming Dice scores.
- **Recommended Probabilities**: p=0.2-0.5 for most; p=1.0 for Resize/Normalize. Balance via SomeOf (e.g., apply 2-3 of 5 color transforms).
- **Edge Preservation**: Use NEAREST interpolation for masks; prefer CLAHE/RandomGamma over blurs; Affine as Elastic alternative (preserves sharpness).

### 3. Human/Person Segmentation Specific
- **Strategies**: Focus on proportion-preserving: HorizontalFlip (p=0.5), Affine (low shear/scale), Perspective (mild viewpoint). For poses: low-rotate ShiftScaleRotate (±15°). Augment with synthetic poses via copy-paste (if instance seg).
- **Preserve Proportions**: Affine/ShiftScaleRotate with scale_limit=0.1-0.2; avoid elastic warps.
- **Varied Poses/Anatomy**: OneOf for pose simulation: `A.OneOf([A.Perspective(p=1.0), A.Affine(shear=5, p=1.0)], p=0.3)`. Use transfer learning from COCO humans.

### 4. Color Augmentation
- **Mixed Color/B&W**: Randomly apply ToGray (p=0.2) on color images; HSV with sat_shift=0 for B&W simulation. HueSaturationValue: hue=10-20, sat=20-40, val=10-20.
- **Robustness**: ChannelShuffle (p=0.1); RGBShift for desaturation. Train on mixed to handle post-processing effects.

### 5. Lighting/Contrast Augmentation
- **Extreme Lighting**: CLAHE (clip=2-4) for silhouettes/backlit; RandomGamma (80-120) for dynamic range.
- **Parameters**: RandomBrightnessContrast: brightness/contrast_limit=0.3; apply post-Resize. For HDR: combine with RandomGamma p=0.3.

### 6. Edge Preservation
- **Blur vs Preserve**: Blurs (Gaussian/Motion) erode edges—avoid. CLAHE/RandomGamma enhance without blurring.
- **Elastic Alternatives**: Affine (shear=5-10) or OpticalDistortion (low limit=0.1)—less warp, sharper boundaries.
- **Maintain Sharpness**: Always NEAREST for masks; post-aug sharpen with IAASharpen (p=0.1) if needed.

### 7. Kaggle Competition Strategies
- **Winning Pipelines**: From salt segmentation (Kaggle): Compose with Crop, Flip, BrightnessContrast, GaussNoise (p=0.5 each). Deepfake winner: Heavy OneOf for blurs/noise. Common: Start with geometric (p=0.5), add color (p=0.3), noise (p=0.2). Use SomeOf for 2-3 variants.
- **Patterns**: Incremental addition; test val Dice after each. Top solutions: Albumentations for speed, nested OneOf for balance.
- **Tips**: Monitor worst-case Dice; augment minorities (e.g., silhouettes via CLAHE).

### 8. Albumentations Compose Strategies
- **OneOf/SomeOf**: OneOf: Pick 1 from group (e.g., noise types, p=0.3). SomeOf: Pick k from n (e.g., 2 of 4 colors, k=2, p=0.4).
- **Ordering**: Geometric first (Resize/Crop/Flip), then color/intensity (Brightness/CLAHE), noise last, Normalize at end. E.g., `Compose([A.Resize(...), A.HorizontalFlip(...), A.OneOf([...]), A.Normalize(...)])`.
- **Probability Balancing**: Total effective p = product (e.g., Compose p=0.9 * inner OneOf p=0.5 = 0.45). Use weights in OneOf for imbalance.

### 9. Domain-Specific Recommendations
- **Portrait/Fashion**: HueSaturationValue for effects; CLAHE for studio lighting; low Perspective for poses. Augment backgrounds separately (e.g., RandomErasing p=0.1).
- **Pro vs Amateur**: GaussNoise (low) for grain; RandomGamma for exposure diffs.
- **Varied Backgrounds**: CopyPaste (p=0.2) to swap complex bgs (nature/urban).

### 10. Performance Considerations
- **Expensive Transforms**: Elastic/GridDistortion (O(n^2) grid ops); Perspective/Affine (warp-heavy). Blur/MotionBlur also slow.
- **Optimization**: Use OpenCV backend (cv2); avoid PIL. For speed: num_workers=4-8 in DataLoader; batch aug on CPU, norm on GPU. Albumentations is 2-10x faster than imgaug/torchvision on CPU—use for bottlenecks.
- **GPU vs CPU**: CPU-primary (Albumentations optimized); GPU via Kornia for batches. Hybrid: CPU aug + GPU norm.

### Recommended Augmentation Pipeline for Human Segmentation
Based on your dataset (upright people, varied lighting/color, complex bgs, low noise), here's a tailored train_transform. Removes issues (VerticalFlip aggressive rotate, Elastic, heavy noise); adds CLAHE/Gamma for lighting, ToGray for B&W, mild Affine. Uses OneOf/SomeOf for balance. Expected: Improves worst-case Dice by focusing on realism.

```python
import albumentations as A
from albumentations.pytorch import ToTensorV2

train_transform = A.Compose([
    A.Resize(384, 384),  # Always
    A.OneOf([  # Geometric: Proportion-preserving
        A.HorizontalFlip(p=1.0),
        A.Affine(translate_percent=0.1, scale=(0.9, 1.1), shear=5, p=1.0),  # Elastic alt
    ], p=0.5),
    A.SomeOf([  # Color/Lighting: 2 of 4
        A.RandomBrightnessContrast(brightness_limit=0.3, contrast_limit=0.3, p=1.0),
        A.CLAHE(clip_limit=3.0, tile_grid_size=(8,8), p=1.0),  # Silhouettes
        A.RandomGamma(gamma_limit=(80, 120), p=1.0),  # Exposure
        A.HueSaturationValue(hue_shift_limit=10, sat_shift_limit=20, val_shift_limit=10, p=1.0),  # Color effects
    ], n=2, p=0.4),
    A.OneOf([  # Noise: Light for ~3% grain
        A.GaussNoise(var_limit=(10, 20), p=1.0),
        A.ISONoise(intensity=(0.05, 0.1), p=1.0),
    ], p=0.1),  # Low p
    A.ToGray(num_output_channels=3, p=0.2),  # B&W mix
    A.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]),
    ToTensorV2(),
], p=1.0)  # Full pipeline always runs
```

Val transform: Just Resize + Normalize (no aug).

### Transforms to DEFINITELY REMOVE from Current Pipeline
- **VerticalFlip (p=0.3)**: People aren't upside-down—distorts upright poses, hurts worst-case.
- **ShiftScaleRotate rotate_limit=30**: Too aggressive; reduce to 15 or replace with Affine (shear-limited).
- **ElasticTransform**: Distorts bodies, blurs edges—major Dice dropper for humans.
- **GaussNoise (p=0.2)**: Dataset has ~3% noise; high p over-augments clean images, adds irrelevant variance.

### Transforms to ADD Based on Dataset Characteristics
- **CLAHE (p=0.3)**: For extreme lighting/silhouettes—enhances contrast without blur.
- **RandomGamma (p=0.2)**: Simulates post-processed effects/exposure.
- **ToGray (p=0.2)**: Handles B&W mix; replicate to 3 channels.
- **HueSaturationValue (p=0.3)**: For color variations/effects.
- **Affline (p=0.3)**: Mild warp as Elastic alt—preserves proportions.

Test incrementally: Add one, retrain 50 epochs, check min Dice. Expected gain: +0.05-0.10 on worst-case via targeted lighting/edge focus.

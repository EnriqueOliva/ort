Funciones de Activación - Explicadas en Clase

Las funciones que el profesor explicó el 01-09-2025

1. Función Signo (la primera)

sign(z) = +1 si z > 0
sign(z) = -1 si z < 0
Época: Rosenblatt, 1950s
Problema: Derivada = 0 en todos lados
No funciona con gradient descent

2. Tanh (años 90)

tanh(z)
Rango: [-1, 1]
Usada en: LeNet (1989/1998)
Problema: SATURACIÓN
Cuando z es muy grande/chico, derivada → 0

3. Sigmoid (años 90)

sigmoid(z) = 1 / (1 + e^(-z))
Rango: [0, 1]
Usada para: Probabilidades
Problema: SATURACIÓN
Mismos problemas que tanh

4. ReLU (la más usada)

ReLU(z) = max(0, z)
Si z < 0 → 0
Si z > 0 → z
Ventaja: NO hay saturación
Derivada: 1 o 0 (súper simple)
El profesor: "mucho más fácil de entrenar"

5. Leaky ReLU

LeakyReLU(z) = z si z > 0
LeakyReLU(z) = α·z si z ≤ 0
(α = 0.01 o 0.02)
Ventaja: No mata completamente los negativos
Los multiplica por un número chiquito

6. SoftPlus (usada en clase)

softplus(z) = log(1 + e^z)
El profesor repitió: "la derivada de softplus es sigmoid"
Ventaja: ReLU pero suavizada
No tiene el quiebre brusco de ReLU

7. GELU (Transformers)

GELU(z) = z·Φ(z)
(con distribución Gaussiana)
El profesor dijo: "tiene una pancita"
Única: NO es monótona
Usada en Transformers modernos

8. SoftMax (¡ESPECIAL! - Para clasificación)

⚠️ DIFERENTE DE TODAS LAS DEMÁS:
NO trabaja elemento por elemento. Necesita TODO el vector.
softmax(z)ᵢ = e^zᵢ / Σ(e^zⱼ)

Ejemplo: [2.0, 1.0, 0.1] → [0.66, 0.24, 0.10]
(las salidas SUMAN 1.0)
SOLO para: Última capa en clasificación múltiple
Ventaja: Convierte números a probabilidades
Se usa con: Categorical Cross-Entropy
El gráfico muestra: 3 entradas diferentes → 3 probabilidades
Diferencia clave:
• Todas las demás: z → número (independiente)
• SoftMax: [z₁, z₂, z₃] → [p₁, p₂, p₃] (todos juntos)

¿Cuál usar?

El profesor dijo textual: "buscar si ya se ha usado antes, copiar eso y medio adaptarlo"

No hay regla universal. Cada una apareció para resolver algún problema específico.

Funciones de Pérdida (Loss Functions)

Las funciones que determinan qué tan equivocada está tu red

1. Binary Cross-Entropy (BCE)

Loss = -[y·log(ŷ) + (1-y)·log(1-ŷ)]
Para: Clasificación BINARIA (2 clases)
Última capa: 1 neurona + Sigmoid
Output: [0.9] → probabilidad entre 0 y 1
Ejemplo:
¿Es spam?
Real: y = 1 (sí es spam)
Predicción: ŷ = 0.9
Loss = -log(0.9) = 0.105 ✓ (error pequeño)

Si predice ŷ = 0.1:
Loss = -log(0.1) = 2.30 ✗ (error grande)
Cuándo usarla: spam/no spam, perro/gato, sí/no

2. Categorical Cross-Entropy (CCE)

Loss = -Σ(y_i · log(ŷ_i))
Para: Clasificación MULTICLASE
Última capa: N neuronas + Softmax
Output: [0.1, 0.7, 0.2] → probabilidades que suman 1
Ejemplo:
CIFAR-10: [car, bus, truck, motor, person]

Real: truck → y = [0, 0, 1, 0, 0]
Predicción: ŷ = [0.1, 0.05, 0.7, 0.1, 0.05]
Loss = -log(0.7) = 0.357

Solo importa la probabilidad de la clase correcta
Cuándo usarla: CIFAR-10, ImageNet, dígitos (clases excluyentes)

3. Mean Squared Error (MSE)

Loss = (1/N) · Σ(y_i - ŷ_i)²
Para: REGRESIÓN
Última capa: N neuronas SIN activación
Output: [195340.5] → números reales
Ejemplo:
Predecir precio de casa:
Real: y = $200,000
Predicción: ŷ = $195,000
Error: (5,000)² = 25,000,000

Si predice $150,000:
Error: (50,000)² = 2,500,000,000
↑ Penaliza MUCHO más los errores grandes
Cuándo usarla: precios, temperaturas, edades, coordenadas

4. Mean Absolute Error (MAE)

Loss = (1/N) · Σ|y_i - ŷ_i|
Para: REGRESIÓN (menos sensible a outliers)
Última capa: N neuronas SIN activación
Output: [32.5] → números reales
Diferencia con MSE:

Error = $10,000
MSE: 10,000² = 100,000,000
MAE: |10,000| = 10,000

Error = $100,000
MSE: 100,000² = 10,000,000,000 ← Desproporcionado
MAE: |100,000| = 100,000 ← Proporcional
Cuándo usarla: cuando hay outliers, penalizar linealmente

Resumen: ¿Qué Loss usar?

Función Tipo Última Capa Output Cuándo
BCE Clasificación binaria 1 + Sigmoid [0.9] ¿Es perro?
CCE Clasificación multiclase N + Softmax [0.1, 0.7, 0.2] CIFAR-10
MSE Regresión N (sin activación) [195340.5] Precio
MAE Regresión (outliers) N (sin activación) [32.5] Edad
Regla práctica:
• ¿Clasificar en 2 categorías? → BCE + Sigmoid
• ¿Clasificar en N categorías? → CCE + Softmax
• ¿Predecir número? → MSE o MAE (según outliers)
